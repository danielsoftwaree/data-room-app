# Product

## Register

product

## Users

Due-diligence participants at Acme Corp. (deal teams, reviewers, external counsel)
organizing and reviewing documents for a multi-billion dollar acquisition — plus
take-home evaluators clicking through the app. They are in a focused work
session: uploading, structuring, finding and reading PDFs. Desktop-first,
mouse + keyboard.

## Product Purpose

A virtual Data Room MVP: create data rooms, nest folders, upload and view PDF
files, rename/move/delete safely. Success = a reviewer can organize and find
any document quickly, every visible control works, and edge cases (duplicates,
deletes with contents, failed uploads) are handled without confusion.

## Brand Personality

Confident, calm, precise. Discord-inspired visual language (Blurple as the
single action color, deep-indigo dark mode) applied with product restraint —
the tool disappears into the task.

## Anti-references

- Not a marketing site: no hero sections, no decorative motion, no gradients-as-identity.
- Not "AI slop SaaS": no ghost-cards (1px border + wide soft shadow), no
  over-rounded cards, no identical icon-card grids, no eyebrow labels.
- Not a toy demo: no dead controls, no "coming soon" placeholders left visible,
  no lazy plurals ("1 member(s)"), no raw locale leaks.

## Design Principles

1. Every visible control does something real — unimplemented features are cut,
   not badged "Soon".
2. Earned familiarity: Drive/Dropbox-shaped patterns, shadcn component
   vocabulary, one accent color for actions and selection only.
3. States are the product: loading, empty, error, success handled everywhere;
   destructive actions always show impact.
4. Copy is precise: correct plurals, one locale, one terminology
   (Data Room / folder / file) everywhere.
5. Restraint over decoration: density where it helps the task, motion only to
   convey state (150–250 ms, ease-out).

## Accessibility & Inclusion

WCAG AA: body text contrast >= 4.5:1, visible focus indicators, full keyboard
navigation for dialogs/menus/tables, prefers-reduced-motion respected,
touch targets >= 44px where applicable.
